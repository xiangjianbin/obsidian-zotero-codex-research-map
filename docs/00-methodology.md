# 方法论总览

这套方法的核心是把科研综述拆成四层：

1. **科学问题树**：定义领域中真正需要解决的问题。
2. **问题节点**：每个节点记录问题定义、已有方法、代表论文、未解决缺口和机器学习机会。
3. **文献卡片**：每篇文章只负责保存证据，不承担全局结构。
4. **矩阵和图谱**：用 Dataview、Canvas、Mermaid 自动聚合。

## 为什么不用普通文献综述表

普通表格容易变成“作者、年份、方法、结论”的堆叠。问题树更适合科研选题，因为它强迫你回答：

- 这个问题是否已经被传统方法部分解决？
- 剩下的缺口是什么？
- 机器学习是否真的解决缺口，还是只换了一个黑箱？
- 论文证据来自理论、合成数据、实测案例还是 benchmark？

## 推荐笔记类型

| 类型 | 作用 | 例子 |
|---|---|---|
| `research-map` | 总览入口 | `00 重磁反演总览.md` |
| `problem-tree` | 树状图 | `00-问题树.md` |
| `problem` | 科学挑战节点 | `问题节点/结构一致性假设.md` |
| `paper` | Zotero 文献卡片 | `Zotero/文献卡片/fang2025.md` |
| `evidence-matrix` | 文献-问题矩阵 | `01-文献矩阵.md` |
| `ml-opportunity-map` | 机器学习切入点 | `02-机器学习可切入问题.md` |

## 状态字段

问题节点建议使用：

- `status: open`
- `status: partially-solved`
- `status: solved-for-limited-cases`

文献卡片建议使用：

- `status: unread`
- `status: imported`
- `status: pdf-reviewed`

证据级别建议使用：

- `theoretical`
- `synthetic`
- `field-case`
- `benchmark`
- `review`

## 判断 ML 机会的三条线

一个问题适合机器学习，不是因为“能套神经网络”，而是因为：

1. 传统方法有明确瓶颈。
2. 有可构造或可收集的训练/验证信号。
3. 输出可以被物理正演、地质解释、实验观测或不确定性校准审查。
